# Redwood Experience: Move Action Extensibility in Hiring Section of Job Applications and Candidate Profiles

`⚙️ Steps to Enable`

> 📖 [View on Oracle Help Center](https://docs.oracle.com/en/cloud/saas/readiness/hcm/26c/recr-26c/26C-recruiting-wn-f43488.htm)

You can configure the fields in the Move application panel when the action is performed from the Hiring section of the Activity tab, which is available in job applications and candidate profiles. These fields can be configured to be hidden and required:

• Phase
• State
• Comment
• Reason (the field is only visible based on the candidate selection process configuration)

Move Application Panel

**Business benefit:** You can display or hide fields from users to align with your business processes.

## ⚙️ Steps to Enable and Configure

You can control the display of fields in the Move application panel. In Visual Builder Studio, go to Hiring > Job Application > Move Job Application Action and create business rules. For details, see How do I hide or show a field in Visual Builder Studio?

Note that when you configure the Move Job Application Action on the candidate profile's Hiring section, the job application phase and state conditions aren't available to include in your rules.

Fields Configuration in Visual Builder Studio

To use Redwood experience, you need to enable the Recruiting Redwood pages profile options. For details, see Set profile options for Redwood pages

For details on enabling and customizing Redwood for HCM, see How do I adopt Redwood for HCM? and Extending Redwood Applications for HCM and SCM Using Visual Builder Studio
