# Redwood Experience: Move and Continue Action When Moving a Job Application

`⚙️ Steps to Enable`

> 📖 [View on Oracle Help Center](https://docs.oracle.com/en/cloud/saas/readiness/hcm/26c/recr-26c/26C-recruiting-wn-f43884.htm)

When the Move and Continue action is enabled on the job applications list or job application details page, you can move a job application multiple times consecutively.

For example, on the job applications list, select a job application then select the **Move Application** action. On the Move application panel, select a phase and state then select **Move and Continue**. The application is moved to the selected phase and state and you can continue to move the application.

**Note:** The Move and Continue action is only available when moving one job application. When you select multiple applications in the list, the Move and Continue action is disabled.

Move and Continue Action

**Business benefit:** Users save time by quickly progressing a job application through the selection process in a single move action.

## ⚙️ Steps to Enable and Configure

You can display the Move and Continue action on the job applications list and job application details page. In Visual Builder Studio, go to Hiring > Job Application List > Move Application or Hiring > Job Application > Move Application and make the **Move Again Action** field visible by creating a business rule.  For details, see How do I hide or show a field in Visual Builder Studio?

Move Again Action in Visual Builder Studio

To use Redwood experience, you need to enable the Recruiting Redwood pages profile options. For details, see Set profile options for Redwood pages.

For details on enabling and customizing Redwood for HCM, see How do I adopt Redwood for HCM? and Extending Redwood Applications for HCM and SCM Using Visual Builder Studio
