# Pay Analyst Enhanced with Payroll Policy Guidance

`⚙️ Steps to Enable` · `💡 Tips & Considerations` · `🔐 Access Requirements`

> 📖 [View on Oracle Help Center](https://docs.oracle.com/en/cloud/saas/readiness/hcm/26c/payr-26c/26C-payroll-wn-f50493.htm)

The **Pay Analyst** has been enhanced to combine its existing payslip information capabilities with those of the **Pay Policy Advisor**, providing employees with a single AI-powered assistant for payroll-related questions.

In addition to providing secure access to payslip details, the enhanced Pay Analyst can now answer employees' questions about payroll policies, including details on payroll processes, payroll schedules, and other payroll-related topics using your organization's payroll policy documents.

Improve the employee self-service experience with a single AI-powered assistant for payslip information and payroll policy guidance. Employees can quickly access payslip information and receive consistent, organization-specific payroll policy answers through a conversational interface, reducing the need to switch between multiple assistants.

## ⚙️ Steps to Enable and Configure

• Your environment must have the appropriate services for Oracle Applications Platform deployed. For more information, see FAQ2521 on My Oracle Cloud Support
• Set the **Enable Security Console External Application Integration** (**ORA_ASE_SAS_INTEGRATION_ENABLED**) profile option to **Yes**, and enable permission groups for the appropriate roles. See Access Requirements for AI Agent Studio
• To learn how to set up AI agents for Redwood pages, see Create AI Agents Using Preconfigured Agent Team Templates

You need to configure security to use this feature. See the *Access Requirements*section.

## 💡 Tips and Considerations

To enable payroll policy guidance, upload your organization's payroll policy documents to the workflow that includes the Pay Analyst agent.

## 🔐 Access Requirements

You can view all available AI Agents from the AI Agents work area. The agents you can view depend on the roles and privileges assigned to you. To access this agent, your role must be explicitly granted access to it by an AI Studio Administrator.
