# Course Enrollment Primary Action

`⚙️ Steps to Enable` · `💡 Tips & Considerations`

> 📖 [View on Oracle Help Center](https://docs.oracle.com/en/cloud/saas/readiness/hcm/26c/lear-26c/26C-learning-wn-f50114.htm)

Learning specialists can make Enroll the primary action on course details pages.

When multiple offerings are available, selecting Enroll opens the Find Offerings page so learners can select an offering, event, or self-paced learning.

When only a single self-paced offering is available, selecting Enroll enrolls learners directly if approval isn't required. If approval is required, selecting Enroll starts the enrollment approval workflow.

Provide a more consistent enrollment experience by making Enroll the primary action on course details pages while preserving existing offering-selection and approval workflows.

## ⚙️ Steps to Enable and Configure

This feature is disabled by default.

1. In Visual Builder Studio, set the Show Enroll on Primary Actions Menu for Course Details Pages page property to **True**.

## 💡 Tips and Considerations

• This change applies only to course details pages. Details pages for other learning types aren't changed.
• The existing course-level request workflow isn't affected by this page property setting.
