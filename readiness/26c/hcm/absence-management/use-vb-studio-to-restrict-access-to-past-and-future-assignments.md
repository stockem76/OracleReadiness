# Use VB Studio to Restrict Access to Past and Future Assignments

`⚙️ Steps to Enable`

> 📖 [View on Oracle Help Center](https://docs.oracle.com/en/cloud/saas/readiness/hcm/26c/amg-26c/26C-absence-mgmt-wn-f49550.htm)

Absence Management enables users to record absences for past assignments that concluded within the last year, as well as, for all active future assignments. You can now further refine these options using two new page properties in the Visual Builder Studio.

Page Property | Description
Future Assignment Flag | If you set this property value to N, the application restricts workers from entering absences for future assignments.
Assignment Effective Days | If you assign a value less than 365 days, the application limits absence entry for past assignments to only those assignments that ended within the specified number of days.

These new properties are available on the New Absence, Absence Bid and Redwood Calendar pages. The restrictions apply only to employees and managers. HR Specialists and absence administrators remain unaffected.

Exercise greater control over when employees and managers can record absences for past or future assignments. Helps enforce business rules more precisely while keeping HR and admin flexibility intact.

## ⚙️ Steps to Enable and Configure

You don't need to do anything to enable this feature.
