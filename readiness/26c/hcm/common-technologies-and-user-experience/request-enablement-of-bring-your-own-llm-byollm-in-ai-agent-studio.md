# Request Enablement of Bring Your Own LLM (BYOLLM) in AI Agent Studio

`⚙️ Steps to Enable` · `🔐 Access Requirements`

> 📖 [View on Oracle Help Center](https://docs.oracle.com/en/cloud/saas/readiness/common/26c/common26c/26C-common-wn-f50064.htm)

You can now request enablement of additional supported LLMs in AI Agent Studio. If a supported model from an available provider doesn’t appear in the model selection list while adding LLMs, submit a service request through My Oracle Cloud Support to request its availability. When creating the request, provide the supported provider name as displayed in AI Agent Studio, the requested model and its version, the business use case, and the required timeline. Each request is reviewed, and an estimated availability timeline is provided based on the technical complexity of the requested model.

Most requests are completed within two weeks, while more complex model integrations may require up to four weeks.

**Note**: Enablement service requests can be submitted for supported models through the Bring Your Own LLM (BYOLLM) option when Oracle Cloud Infrastructure is selected as the model provider. However, this process doesn't apply to basic and premium models provided by Oracle that are available for selection, such as GPT-4.1 Mini, GPT-5 Mini.

This feature provides organizations with greater flexibility to adopt the LLMs that best align with their business requirements while maintaining a governed and supported integration process. By enabling organizations to request additional supported LLMs, they can accelerate AI adoption, expand model choice, and align AI Agent Studio deployments with evolving business and industry-specific needs. The streamlined request process helps ensure predictable onboarding timelines while maintaining platform reliability and compatibility standards.
