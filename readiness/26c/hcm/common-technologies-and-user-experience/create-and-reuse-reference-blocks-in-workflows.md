# Create and Reuse Reference Blocks in Workflows

`⚙️ Steps to Enable` · `🔐 Access Requirements`

> 📖 [View on Oracle Help Center](https://docs.oracle.com/en/cloud/saas/readiness/common/26c/common26c/26C-common-wn-f50150.htm)

You can now create reference blocks to define reusable logic within a workflow. These reference blocks can include all available artifacts such as agents, LLMs, tools, and so on. You can then use the same reference block multiple times within your workflow.

To create a reference block, add a node of type Reference Block in your workflow and include the necessary artifacts. To associate the reference block with your workflow, create a node of type Referenceandadd your reference block to it.

Reference blocks help organizations standardize and reuse common logic within workflows. This capability reduces duplication, simplifies flow maintenance, and promotes consistency by enabling the definition of shared functionality. By centralizing commonly used logic in reusable reference blocks, organizations can accelerate workflow development, improve maintainability, and reduce the effort required to update and govern business processes.
