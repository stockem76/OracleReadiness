# Additional Assignment Attributes for Page Properties in Redwood Employment Processes

`⚙️ Steps to Enable` · `💡 Tips & Considerations`

> 📖 [View on Oracle Help Center](https://docs.oracle.com/en/cloud/saas/readiness/hcm/26c/hure-26c/26C-hr-wn-f47244.htm)

Page Properties rules for Redwood Employment pages can now use four additional assignment attributes: User Person Type, Union, Collective Agreement, and Bargaining Unit. These attributes help customers configure guided journeys, header values, buttons, and other page properties should appear in supported flows.

Do not use these new assignment attributes to control the visibility of sections in the employment process.

Additional attributes to configure Employment pages

This enhancement gives customers more precise control over the Redwood Employment experience. Customers can tailor AI Agent visibility, guided journeys, header content, and other page properties based on assignment context, improving relevance and reducing unnecessary page elements for users.

## 💡 Tips and Considerations

For employment update processes, until the user reaches the Assignment section, the values for these fields will be whatever values are there in the current assignment. If the user modifies these values, then the new values are considered****
