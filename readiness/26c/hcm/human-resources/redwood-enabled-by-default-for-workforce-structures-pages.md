# Redwood Enabled by Default for Workforce Structures Pages

`⚙️ Steps to Enable` · `🔐 Access Requirements`

> 📖 [View on Oracle Help Center](https://docs.oracle.com/en/cloud/saas/readiness/hcm/26c/hure-26c/26C-hr-wn-f49852.htm)

Effective from release 26C, the following Redwood workforce structures pages will be enabled by default:

• Grade Rates
• Grades
• Legal Entity HCM Information
• Legal Reporting Unit HCM Information
• Reporting Establishments

The corresponding responsive or classic pages will no longer be available by default. However, you can still access them by manually setting the relevant profile option to **No**. You can see the list of profile options for all these pages in the **Steps to Enable and Configure** section of this What's New.

By using these new pages, you can take advantage of the cohesiveness through the application.

## ⚙️ Steps to Enable and Configure

If you want to, you can disable the profile options for the Redwood pages where they're enabled. You can do so by entering **No**in the **Profile Value** field. Follow these steps: How do I enable a profile option?

This table lists the profile option codes for Redwood Workforce structures pages.

Redwood Page | Profile Option Code
Grades | ORA_PER_GRADES_REDWOOD_ENABLED
Grade Rates | ORA_PER_GRADE_RATES_REDWOOD_ENABLED
Legal Entity HCM Information | ORA_PER_LEGAL_ENTITY_HCM_INFO_REDWOOD_ENABLED
Legal Reporting Unit HCM Information | ORA_PER_LEGAL_REPORTING_UNIT_HCM_INFO_REDWOOD_ENABLED
Reporting Establishments | ORA_PER_REPORTING_ESTABLISHMENTS_REDWOOD_ENABLED

## 🔐 Access Requirements

The roles that had access to the responsive and classic pages will have access to the corresponding Redwood page as well.
