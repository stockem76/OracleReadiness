# Translation Editor on Redwood Extended Lookup Codes Page with Generative AI

`⚙️ Steps to Enable` · `💡 Tips & Considerations` · `🔐 Access Requirements`

> 📖 [View on Oracle Help Center](https://docs.oracle.com/en/cloud/saas/readiness/hcm/26c/hure-26c/26C-hr-wn-f44458.htm)

Generative AI in the translation editor enhances the translation experience in the Redwood Extended Lookup Codes page by leveraging artificial intelligence to help users generate and refine localized content directly within the editor.

With this feature, administrators and content authors can quickly produce accurate translations for extended lookup code attributes across multiple languages, reducing the need for manual translation efforts. This helps improve consistency, speed, and quality of localized content while allowing users to review and edit translations before publishing.

## ⚙️ Steps to Enable and Configure

• Ensure that you have access to Visual Builder Studio (VB Studio). Instances and workspaces should have been correctly set as well.
• The Generate button isn't displayed by default. To display the button, you need to enable the **Show AI Assist** VB constant in the Extended Lookup Codes pages. The VB constant is enabled when you set its value to true by using VB Studio. For more information, see this topic: How do I control the display of a UI element in Visual Builder Studio?
• You need to configure security to use this feature. See the Access Requirements section.

## 💡 Tips and Considerations

• This capability is available only in the Redwood experience.
• Only those text attributes that have a value will appear in the translation panel drawer.
• If you no longer require the translated values, you can update the translated text with the base language text.
• The translations that you generate using Gen AI may contain inaccuracies. You need to review them before you update them.

## 🔐 Access Requirements

You must be granted the Application Administrator role to access VB Studio.
