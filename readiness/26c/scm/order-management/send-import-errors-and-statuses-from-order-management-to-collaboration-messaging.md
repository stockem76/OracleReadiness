# Send Import Errors and Statuses from Order Management to Collaboration Messaging

`⚙️ Steps to Enable` · `💡 Tips & Considerations`

> 📖 [View on Oracle Help Center](https://docs.oracle.com/en/cloud/saas/readiness/scm/26c/order26c/26C-order-mgmt-wn-f46415.htm)

You can now use Collaboration Messaging to view Oracle Order Management failure or error notifications for inbound PO messages submitted via Collaboration Messaging.

**Get these benefits**

• View failure or error notifications on purchase orders, order changes, and cancellations.
• See final processing outcomes after import scheduled processes run.
• Get end-to-end status visibility.

**How it works**

With this new feature the log is updated with the actual final outcome and error details, ensuring the page reflects the true downstream processing result giving you end-to-end visibility. Previously, a transaction would be marked as a success when the inbound message was staged, even if the business processing failed at a later stage.

The failure or error notifications are sent after the Import Sales Orders scheduled process runs. If there are validation or processing errors they will continue to be seen in the log file of the scheduled process.The real value-add now is the visibility of order errors in Collaboration Messaging.

If an error generated by theImport Sales Orders scheduled process includes a sales order line reference, Collaboration Messaging displays it as a line-level error in the sales order line error description. If no line reference is included, Collaboration Messaging displays it as a header-level error.

Error messages for sales orders created in draft status are displayed in the Messaging History page.

If multiple messages reference the same source order number due to revisions or duplicate order creation for example, the system applies the error update to themost recently created message.

## ⚙️ Steps to Enable and Configure

You don't need to do anything to enable this feature.

## 💡 Tips and Considerations

• If the **FOM_IMPORT_VIA_REST_BACKEND** profile option is enabled, Order Management logs all errors during the Import Order process. If it isn’t enabled, the import stops at the first error, so you may not see all errors in a single run. This is the same as the existing order management functionality.  If you don't have that profile enabled you'll need to enable it to use this feature.
• Successful processing does not generate notifications. Visibility into successful processing continues to rely on existing Collaboration Messaging event/acknowledgement functionality when it is configured.
